## üö¶ How to Get Started

### 1. Reporting Bugs
Before creating a bug report, please check the **Issues** tab to see if the problem has already been reported. If not, open a new issue and include:
* A clear, descriptive title.
* Steps to reproduce the issue.

### 2. Mirror Related Issues
If you encounter issues with the mirror (downloads failing, slow speeds, or synchronization errors), please contact **the project lead** directly by opening an issue and tagging it as `infrastructure`.

### 3. Suggesting Enhancements
Have a great idea? We want to hear it!
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful.
* For UI/UX changes, please mention **Miles Kaminsky (Design)** in the thread.

## üõ†Ô∏è Development Workflow

### Pull Requests (PRs)
1. **Fork the Repo:** Create your own copy of the project.
2. **Branching:** Create a branch for your feature (`git checkout -b feature/AmazingFeature`).
3. **Commits:** Write clear, concise commit messages.
4. **Testing:** Ensure your code doesn't break existing features.
5. **Submit:** Open a PR against the `main` branch.

### Review Process
All PRs must be reviewed by the relevant maintainer:
* **Core Code/Architecture:** Reviewed by **Kenneth Daily**.
* **UI/Visuals:** Reviewed by **Miles Kaminsky**.
* **Mirror:** Reviewed by **Logan Patullo**.
